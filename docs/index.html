The Governance Thesis: Prompts as Constitutions

System prompts in AI-assisted developer tools are more than just task instructions; they function as implicit constitutions. These hidden strings of text define the fundamental laws of an agent—what it is, what it can see, who it must obey, and when it must stop.

This research moves beyond the view of system prompts as mere "instruction sets" or "persona definitions." Instead, we treat them as governance artifacts—the foundational legal framework of an AI agent. These prompts encode the distribution of power, the boundaries of visibility, and the conditions for termination.

The Forensic Approach

By capturing raw HTTP traffic from tools like GitHub Copilot and Codex, we extract the underlying system prompts and normalize them into a structured, comparable format. We map out the actual governance layers:

  • Authority Who has the final say? (User vs. Policy vs. Model)
  • Scope What parts of the workspace are visible or writable?
  • Mediation How are tools gated and sequenced?
  • Termination What triggers a hard stop or a "stop-and-ask"?

Architectural Classes of AI Assistants

Suggestion Engines

Minimal authority, focused on low-risk text generation with high policy compliance (e.g., Copilot Inline).

Command Executors

High authority over the terminal, governed by strict procedural rules and "stop-and-ask" triggers (e.g., Codex Exec).

Workspace Agents

Broad authority over the filesystem and tools, managed through mode-tiered permissions (e.g., VS Code Copilot Agent).

Constitutional Stewards

Specialized modes where the agent's primary role is to enforce a specific schema or workflow (e.g., Codex Review).

The Authority Gradient

Policy-Supremacy
Hard-coded refusals and identity disclosures that the model cannot override.
User-Consent
Authority is expanded conditionally through approval-gated escalation.
Model-Autonomy
The model is the final decision-maker within its sandbox, trading consent gates for internal controls.

Prompt Governance Primitives (PGPs)

Modular, atomic control structures that appear across different assistants. These are the building blocks of predictable agent behavior.

PGP-004 Stop-on-unexpected-change

A circuit breaker that triggers a hard stop if the workspace changes unexpectedly.

PGP-001 Approval-gated escalation

A mechanism to request higher privileges only when a specific tool call is blocked.

PGP-009 Identity Disclosure

Hard-coded rules for how an agent must identify itself and its underlying model.
